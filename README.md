# The Data Governor

Practitioner blog for [thedatagovernor.com](https://thedatagovernor.com) — data governance, metadata management, data strategy, and enterprise data programs.

Migrated from WordPress (.info) to Astro SSG + Cloudflare Pages in April 2026.

---

## Stack

| Layer | Technology |
|-------|-----------|
| Framework | [Astro](https://astro.build/) 3.6.4 (SSG) |
| Content | Astro Content Collections (Markdown + Zod schema) |
| Hosting | Cloudflare Pages |
| CDN / DNS | Cloudflare |
| Analytics | Google Analytics 4 (`G-H3M19HMVQE`) |
| Fonts | Google Fonts — Roboto + Roboto Condensed (async loaded) |

> **Note:** Astro 4.x is blocked on this machine by Windows WDAC policy (Rollup 4 requires a native `.node` binary). Stay on Astro 3.6.x until that policy changes.

## Deployment

Deployed manually via Wrangler CLI to Cloudflare Pages:

```bash
npm run build
npx wrangler pages deploy dist --project-name thedatagovernor --branch main
```

The project is not connected to Git auto-deploy — deployments are triggered manually.

**Pages project:** `thedatagovernor` → `thedatagovernor.pages.dev`
**Custom domains:** `thedatagovernor.com`, `www.thedatagovernor.com`

---

## DNS & Redirects

- **Registrar:** Namecheap
- **DNS:** Cloudflare (nameservers: `jaime.ns.cloudflare.com`, `maria.ns.cloudflare.com`)
- **`.com` zone:** Active on Cloudflare. Two CNAME records point `@` and `www` to `thedatagovernor.pages.dev`.
- **`.info` → `.com` redirect:** A Cloudflare Worker (`datagovernor-redirect`) is deployed on the `.info` zone with route `*thedatagovernor.info/*`. It issues a 301 to the matching path on `.com`, preserving SEO equity from the legacy WordPress site.

## Content Schema

Blog posts live in `src/content/blog/*.md` with this frontmatter:

```yaml
title: "Post Title"
description: "Meta description"
pubDate: 2024-01-15
updatedDate: 2024-02-01   # optional
author: "Clinton Munkres"
categories: ["Data Governance"]
tags: ["metadata", "strategy"]
wpSlug: "original-wordpress-slug"  # preserves WP URL if different from filename
image: /2024/01/image-filename.jpg
imageAlt: "Image description"
ogImage: /2024/01/og-image.jpg     # optional, falls back to image
draft: false
```

Post URLs are served from the root (`/post-slug/`) to match the original WordPress URL structure exactly.

## SEO

- **Schema.org JSON-LD:** Every post includes an `@graph` with `Article`, `BreadcrumbList`, and `FAQPage` (auto-parsed from `## FAQ` / `## Frequently Asked Questions` sections in post body, up to 10 pairs).
- **Homepage:** `WebSite` + `Organization` schema.
- **About page:** `Person` schema for Clinton Munkres.
- **Sitemap:** `/sitemap.xml` — all posts, static pages, and category pages.
- **RSS:** `/rss.xml` with redirect from `/feed/`.
- **Canonicals:** All include trailing slash.
- **Google Search Console:** `.com` property verified, sitemap submitted, Change of Address filed from `.info`.

## Dark Mode

Implemented via CSS custom properties with `[data-theme="dark"]` overrides. Theme is stored in `localStorage` and applied before first paint via an inline `is:inline` script in `<head>` to prevent FOUC.

Toggle button is in the masthead (moon/sun SVG icons).

## Images

WordPress media archives were extracted from zip files by year (`2022–2025`) into `public/`. Each image has WebP variants generated by the WordPress site with the naming convention `{original-filename}.webp`. Blog post hero images use a `<picture>` element that serves the `.webp` source with the original as fallback.
